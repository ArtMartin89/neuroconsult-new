# **Детальное Техническое Задание (ТЗ) на разработку Landing Page NEUROCONSULT**

На основе: Next.js \+ TypeScript \+ Tailwind CSS  
Цель: Создание высококонверсионного, минималистичного лендинга в стиле Dark Mode, ориентированного на сегмент SMB.

## **1\. Общие сведения и Принципы Разработки**

| Параметр | Значение | Комментарий |
| :---- | :---- | :---- |
| **Проект** | Лендинг "NeuroConsult: Системный рост бизнеса" | Одностраничный сайт (Landing Page) |
| **Срок** | 10 рабочих дней (Front-end) \+ 2 рабочих дня (API/Интеграция) | Срок может быть скорректирован после утверждения дизайна. |
| **Стек** | Next.js (App Router), TypeScript, Tailwind CSS, lucide-react | Строгое следование стеку. |
| **Дизайн-принцип** | **Минимализм и Высокий Контраст (Dark Mode)** | Референс: https://neuroconsult.by/. Чистые линии, акцент на типографике. |
| **Оптимизация** | Core Web Vitals (LCP \< 2.5s) | Обязательная оптимизация изображений и шрифтов. |

## **2\. Спецификация Дизайна и Стилизации (Tailwind CSS)**

### **2.1. Цветовая Палитра**

| Назначение | Цвет/Класс (Hex) | Описание |
| :---- | :---- | :---- |
| **Фон (Background)** | \#0C0C0C или slate-950 | Глубокий темный фон. |
| **Текст Основной** | \#E5E5E5 или slate-100 | Высококонтрастный, основной текст. |
| **Акцент (CTA, Border)** | \#007BFF (Синий) или blue-600 | Основной акцентный цвет для кнопок, иконок и выделений. |
| **Вторичный Акцент** | \#333333 или slate-800 | Цвет для карточек, форм, секций, чтобы отделить их от основного фона. |
| **Негативный/Ошибка** | red-500 | Сообщения об ошибках формы. |

### **2.2. Типографика**

* **Шрифт:** Inter (или другой современный Sans-serif, загружаемый через Next/font).  
* **Заголовки (H1/H2):** Должны быть крупными, жирными (semi-bold/bold), с увеличенным межстрочным интервалом на Desktop для премиального вида.  
* **Текст:** Чистый, читабельный, стандартный размер.

### **2.3. Иконки**

* Использовать только библиотеку **lucide-react** или кастомные SVG.  
* Стиль: **Lineal (тонкие линии)**.  
* Цвет: Акцентный синий (blue-600) или белый (slate-100) на темном фоне.

## **3\. Компонентный Состав и Структура (Next.js)**

Проект должен быть структурирован по модульному принципу:

| Директория/Файл | Описание |
| :---- | :---- |
| app/page.tsx | Основной файл страницы, импортирующий все секции/блоки. |
| components/sections/ | Компоненты для каждого логического блока (HeroBlock, PainPoints, FAQSection и т.д.). |
| components/ui/ | Переиспользуемые, стилизованные компоненты (Button, Input, Card, Accordion, Counter, etc.). |
| app/api/contact/route.ts | Next.js API Route для обработки отправки формы. |

## **4\. Требования к Блокам и Компонентам (Content & Implementation)**

### **БЛОК 1: Первый экран (Hero Block)**

* **Компонент:** HeroBlock.tsx  
* **Верстка:** Заголовок (H1) и подзаголовок должны быть центрированы или выровнены по левому краю с максимальной шириной (max-w-4xl) для читаемости.  
* **Визуализация:** Использовать **абстрактную, анимированную, геометрическую SVG-схему** . Графика должна быть легкой, не загружающей LCP.  
* **CTA:** Кнопка "Получить бесплатную диагностику..." ведет к БЛОКУ 8\. Стиль кнопки: **Акцентный синий фон, белый текст, скругленные углы, легкий эффект наведения (уменьшение прозрачности или легкий сдвиг).**

### **БЛОК 2: Идентификация боли (Узнавание)**

* **Компонент:** PainPointsSection.tsx  
* **Верстка:** Сетка (Grid) 2x3 на Desktop, 1x6 на Mobile.  
* **Карточки:** Компонент PainCard.tsx.  
  * Стиль: Фон slate-800 или slate-900, тонкая синяя рамка.  
  * UX: Плавный, едва заметный эффект сдвига вверх или изменения цвета рамки при наведении.

### **БЛОК 3: Решение и Уникальность (CVP)**

* **Компонент:** CVPSection.tsx  
* **"Как это работает":** Реализовать в виде 3 вертикальных колонок/карточек, которые на Mobile трансформируются в одну колонку. Используйте иконки для каждого из 3 уровней.

### **БЛОК 4: Процесс внедрения**

* **Компонент:** TimelineSection.tsx  
* **Верстка:** **Вертикальная временная шкала.** Каждый этап должен быть выделен круглой точкой (цвета Акцент) и соединен вертикальной линией.  
* **Контент:** Использовать компонент TimelineStep.tsx для каждого из 5 этапов.

### **БЛОК 5: Экспертиза и команда**

* **Компонент:** ExpertiseSection.tsx  
* **Ключевые факты (Счетчики):** Компонент Counter.tsx.  
  * **Функционал:** Анимация счета от 0 до конечного значения (например, 25, 14, 92\) при попадании блока в зону видимости (Intersection Observer).  
  * **Стиль:** Цифра крупным акцентным цветом (H2), под ней — мелкий, обычный текст-описание.  
* **Медийное лицо:** Использовать компонент Next/image с заполнением (e.g., fill prop) и **Placeholder для фото** .

### **БЛОК 6: Социальные доказательства**

* **Компонент:** CaseStudySection.tsx  
* **Структура:** 4 карточки-кейса. Метрики (цифры) в секции "Результат" должны быть выделены акцентным синим цветом и жирным шрифтом.

### **БЛОК 7: FAQ и снятие возражений**

* **Компонент:** FAQSection.tsx  
* **Функционал:** Использовать компонент **Аккордеон** (например, из Radix UI или Shadcn/ui) для управления видимостью ответов. Только один ответ может быть открыт одновременно.

### **БЛОК 8: Финальный CTA и Форма захвата**

* **Компонент:** ConversionFormSection.tsx  
* **Стиль:** Блок должен быть визуально мощным, возможно, иметь немного другой темный фон для привлечения внимания.  
* **Ограничение/Не подходит:** Выделить эти подблоки тонкой рамкой Акцентного цвета.

## **5\. Требования к Форме и API**

### **5.1. Фронтенд (Валидация)**

* **Телефон:** Маска ввода **\+375 (XX) XXX-XX-XX** (использование React Input Mask).  
* **Email:** Валидация формата (regex).  
* **Все поля:** Обязательны для заполнения.  
* **UX:** Сообщения об ошибках должны быть четкими (цвет red-500) и появляться под соответствующим полем.  
* **Кнопка:** Неактивна, пока все поля не пройдут валидацию.

### **5.2. Next.js API (Backend)**

* **Эндпоинт:** /api/contact (метод POST).  
* **Функционал:**  
  1. Получение и валидация данных.  
  2. Защита от спама (reCAPTCHA v3 \- ключ из .env.local).  
  3. Отправка данных на внешний Webhook (CRM). URL Webhook должен храниться в переменных среды.  
  4. Ответ: 200 OK при успехе, 400 Bad Request при ошибке валидации, 500 Internal Server Error при ошибке CRM/API.

## **6\. Нефункциональные требования**

* **Адаптивность:** Полное соответствие принципам Mobile-First.  
* **SEO:** Обязательная настройка метаданных, включая заголовок, описание и теги Open Graph/Twitter Card.  
* **Аналитика:** Интеграция скриптов Google Analytics 4 и Яндекс.Метрики. Инициализация должна происходить только после согласия пользователя (если требуется) или на уровне компонента/страницы Next.js.  
* **Доступность (A11Y):** Использование семантических тегов HTML5, корректное использование атрибутов aria-label для иконок и нетекстовых элементов.  
* **Кроссбраузерность:** Поддержка всех современных браузеров (Chrome, Edge, Firefox, Safari).
